PKU ACM 3125 Printer Queue

发表于 2016-03-11   |  

用打印机的人很多，需要做排序，有1到9共9种优先级。优先的文件先打印，每一份一分钟。
坑点：如果队列中有比当前文件优先级高，则把他排到队尾。
格式
第1行：运行几次（之后每次运行处理两行数据）
第2.1行：打印队列总数，以及我们的文件所在的位置（位置从0开始）
第2.2行：每个位置上的文件的优先度

时间花费

  1. 做完提交发现这货没有把列表完全按照优先级排序，而是每次有更高优先级的情况时把当前的文件丢到最后。于是把原来 “大于我优先级的总数+同优先级的排序位置”方式放弃

  2. 用了完全模拟的方式，每次读一个，不是就往后丢。只用一个int记录目标文件的序号，如果该需要的文件被打印就输出打印的时间和。

    因为跟踪这个位置的概念很抽象，浪费了一点时间尝试和校准。
  3. 复制数组的时候居然在for里写了[0] XD

题目会给出多种邮票，一次最多买4张，有指定的花费额度。在花完钱的前提下，按以下优先度购买。

  1. 种类多

忽略已经跟踪的文件

某工程project用Git管理代码，但是在他的根目录下有个配置文件，比如project.iws是不需要git每次跟踪它的修改记录的。
一般做法是在.gitignore文件中添加一行

project.iws

但是这个文件如果之前已经被git跟踪了，这样修改是没有用的。每次修改完以后，用git status还是能看到提示它被修改了。
解决办法就是在git中删除这个文件的跟踪记录，用这个命令

这样就从git的跟踪记录中删除了这个文件的跟踪记录。配合之前在.gitignore加的那行配置，以后你修改 project.iws 这个文件，git就不会有提示了。

自己补充
如果要忽略一整个文件夹，跟.gitignore文件的写法一样，比如 git rm --cached Library/ 。
git 提示我需要加-r ，加载末尾就好了

忽略特殊文件

文章介绍了github已经推荐的一些配置文件，针对各种语言、系统容易自动生成的文件做了各种配置。
地址:(https://github.com/github/gitignore)

hexo旧坑，明明做了SSH，为什么每次还会提示我要输入用户名密码

看起来是访问方式的锅，下面是找到的资料

参考这里(http://segmentfault.com/q/1010000000599327)

因为你用的是 https 而不是 ssh。 可以更新一下 origin:

git remote remove origin
git remote add origin git@github.com:Username/Your_Repo_Name.git
注意：如果你的仓库没有add ssh 只是在.ssh上添加了，在你的仓库里还要添加一次这样才会生效。添加步骤同上。可能要等一分钟左右才会生效。

PKU ACM 1009 Edge Detection

发表于 2016-03-09   |  

这次题目都不想贴了，智商退化了之后有点干不过它。
大意是处理一张图像，每个像素点与周围八个比较，取最大的差值作为该位置的结果值。
考点其实是压缩，因为给的数据里有10^9这样的变态值，如果傻不拉唧全部解压开计算内存就爆炸了。

目前的代码提交后提示running time error。我就日了狗了，本地没错啊。
反复设几次return后，确定在新建小3*width数组时有问题。但没找到原因前，又跳了内存超标的错误。

老子解出来很给面子了啊，别在我做回复训练的时候要求太高。Java还要限制10000KB

TODO 等老了之后或许有时间来优化，目测有这样几个点

  1. 检测用的二维数组设定固定长宽，不根据输入内容新建
  2. 点与周围差值比较时新建了8个int，这个应该也可以省
  3. 可以尝试直接从压缩后的结果开始计算，不考虑展开图像后的对比
